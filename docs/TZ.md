## 1. Концепция

**Миссия:** помочь людям контролировать себя и становиться лучше каждый день.

Приложение построено на метафоре **двух сторон**:

|          | **Светлая сторона**             | **Тёмная сторона**                          |
| -------- | ------------------------------- | ------------------------------------------- |
| Суть     | Развитие, рост                  | Отказ от вредного                           |
| Привычки | Книги, спорт, **свои занятия**… | Курение, ногти, соцсети…                    |
| UI       | Ползунки + **помодоро-таймер**    | Тёмный фон, таймеры «чистого времени»       |
| Ввод     | Ползунок: сколько сделал        | Таймер: сколько держусь + кнопка «Сорвался» |

**Ключевые механики:**

- регистрация → светлая сторона (выбор роста) → тёмная сторона (выбор отказа);
- калибровка от **текущего уровня** пользователя (до приложения) + профиль и свободное время;
- ползунки и **помодоро** для полезных привычек (в т.ч. свои: программирование, Blender 3D…);
- прогрессивное усложнение целей;
- 2 пропуска в неделю (только светлая сторона + английский);
- push по расписанию подъёма/сна;
- push при «тупых делах» (TikTok, соцсети) — старт сессии и окончание 15 мин;
- push-подбадривания для некурящих (3–5 раз в день);
- подписка + залог.

**Целевая платформа:** браузерная PWA (Chrome, Safari iOS 16.4+, Firefox, Edge). Нативные приложения — не в scope v1.

**Рынок:** Россия. Платежи в рублях через ЮKassa.

## 2. Профиль пользователя

Данные собираются в два этапа: **регистрация** и **онбординг**.

### 2.1. Регистрация (сразу при входе)

| Поле    | Обязательно                      |
| ------- | -------------------------------- |
| Email   | да                               |
| Пароль  | да                               |
| Имя     | да                               |
| Возраст | да                               |
| Пол     | да (`male` / `female` / `other`) |

### 2.2. Онбординг (после регистрации)

| Поле                         | Обязательно | Зачем                                              |
| ---------------------------- | ----------- | -------------------------------------------------- |
| Вес (кг)                     | да          | Калибровка нагрузки                                |
| Рост (см)                    | да          | Калибровка нагрузки                                |
| Время подъёма                | да          | Push, закрытие дня                                 |
| Время отхода ко сну          | да          | Push, закрытие дня                                 |
| Свободное время в день (мин) | да          | Распределение дневной нагрузки                     |
| Часовой пояс                 | авто        | `Intl.DateTimeFormat().resolvedOptions().timeZone` |

**Часовые пояса РФ:** Россия включает много часовых поясов. Часовой пояс должен определяться автоматически при регистрации и быть редактируемым в профиле. Все уведомления и автозакрытие дня (23:59) считаются в **локальном времени пользователя**.

### 2.3. Дневной бюджет времени

- Целевая нагрузка: **~60 минут в день** на все полезные привычки вместе.
- Если пользователь указал меньше 60 мин свободного времени — бюджет = его значение.
- Если больше — бюджет = 60 мин (не перегружаем).
- Бюджет **делится поровну** между выбранными привычками светлой стороны.
- Пример: 3 привычки, бюджет 60 мин → ~20 мин на каждую → из этого считаются страницы, повторения, секунды планки.

**Важно:** модуль **«Английский» НЕ входит** в дневной бюджет свободного времени. Это отдельная опциональная нагрузка (5–15 минут) и не влияет на расчёт целей спорта/чтения.

**Правила:**

- Изменение профиля **не пересчитывает** текущие цели автоматически.
- Кнопка «Пересчитать цели» — v1.1.

## 3. Калибровка целей

**Главный принцип:** приложение стартует с **реального уровня** пользователя, а не с абстрактных норм.

При онбординге для каждой выбранной привычки спрашиваем: **«Сколько сейчас, до приложения?»**

| Пример                | Поле                   | Единица      |
| --------------------- | ---------------------- | ------------ |
| Курит 20 сигарет      | `baseline_value = 20`  | сигарет/день |
| 3 ложки сахара        | `baseline_value = 3`   | ложки/день   |
| 15 конфет             | `baseline_value = 15`  | штук/день    |
| Уже отжимается 20 раз | `baseline_value = 20`  | раз/день     |
| Читает 5 страниц      | `baseline_value = 5`   | страниц/день |
| Соцсети 2 часа        | `baseline_value = 120` | минут/день   |

Стартовая цель считается из: **baseline + пол + возраст + вес + рост + дневной бюджет времени**.

### 3.1. Стороны, типы и направление прогрессии

| Сторона | Тип          | Направление  | UI               | Пропуски |
| ------- | ------------ | ------------ | ---------------- | -------- |
| Светлая | `target`     | `increase` ↑ | Ползунок         | До 2/нед |
| Тёмная  | `limit`      | `decrease` ↓ | Ползунок + лимит | Нет      |
| Тёмная  | `abstinence` | `abstain`    | Таймер           | Нет      |

Поля в `habits`:

- `baseline_value` — ответ пользователя при онбординге
- `current_goal` — цель на сегодня
- `progression_direction` — `increase` | `decrease` | `abstain`
- `phase` — `reduction` | `abstinence` (для курения: сначала снижение, потом полный отказ)

### 3.2. Светлая сторона — рост от текущего уровня

1. Пользователь вводит **текущий уровень** (`baseline_value`).
2. Система считает **рекомендуемую** цель по бюджету времени и профилю.
3. **Стартовая цель** = `max(baseline_value, recommended)` — не опускаем ниже того, что уже умеет.

| Привычка  | Рекомендация (если baseline не задан) |
| --------- | ------------------------------------- |
| Книги     | ~2 стр/мин × выделенные минуты        |
| Бег       | выделенные минуты                     |
| Отжимания | ~2 сек/повтор                         |
| Планка    | минуты × 60 сек                       |

**Уже делает 20 отжиманий** → старт с **20**, не с 5.  
**Делает 0** → старт по формуле.

**При успехе:** `current_goal += step` (+1 или +5%).  
**При провале:** цель не снижается.

### 3.3. Тёмная сторона — снижение от текущего уровня

| Привычка         | Тип                    | Старт                                       | Логика                                                                                   |
| ---------------- | ---------------------- | ------------------------------------------- | ---------------------------------------------------------------------------------------- |
| **Курение**      | `limit` → `abstinence` | `current_goal = baseline` (напр. 20)        | Фаза 1: ≤ цели → успех → −1 сигарета/день. При `goal = 0` → фаза 2: таймер, полный отказ |
| **Сахар**        | `limit`                | `current_goal = baseline` (напр. 3 ложки)   | Успех → −1 ложка (мин. 0)                                                                |
| **Сладости**     | `limit`                | `current_goal = baseline` (напр. 15 конфет) | Успех → −1 шт (мин. 0)                                                                   |
| **Соцсети**      | `limit`                | `current_goal = baseline` (мин)             | Успех → −5 мин (мин. 15 мин). + сессии «тупых дел»                                       |
| **Грызть ногти** | `abstinence`           | сразу отказ                                 | Таймер + «Сорвался», без числового baseline                                              |

**Пример курения (20 сиг/день):**

```
День 1: цель ≤ 20 → выкурил 18 → успех → завтра цель 19
День 2: цель ≤ 19 → выкурил 19 → успех → завтра цель 18
...
День 21: цель ≤ 0 → фаза abstinence → таймер «не курю»
```

**При провале** (превысил лимит): цель **не повышается** обратно, остаётся на месте.

### 3.4. Прогрессия — сводка

| Сторона    | Успех                   | Провал           | Пропуск          |
| ---------- | ----------------------- | ---------------- | ---------------- |
| Светлая ↑  | `goal += step`          | goal не меняется | goal не меняется |
| Тёмная ↓   | `goal -= step` (мин. 0) | goal не меняется | —                |
| Abstinence | день без срыва          | «Сорвался»       | —                |

Пользователь может вручную изменить цель в настройках привычки.

### 3.5. Свои привычки (светлая сторона)

Пользователь добавляет **любое занятие** сам:

| Примеры | Единица |
|---------|---------|
| Программирование | минуты |
| Blender 3D | минуты |
| Рисование | минуты |
| Изучение Python | минуты / уроки |

**При создании:**

- Название (свободный текст)
- Единица: `minutes` (по умолчанию) / `pages` / `reps` / `lessons`
- `baseline_value` — сколько сейчас в день
- Иконка (опционально)

**Расчёт:** `current_goal = max(baseline, доля бюджета времени)`. При успехе +5 мин или +1 помодоро.

**Помодоро** — основной способ отметки для минутных своих привычек.

**Где добавить:** онбординг (кнопка «+ Своё занятие»), профиль, светлая сторона. Лимит 6 привычек суммарно.

### 3.6. Пропуски (гибкий режим)

Для **полезных** привычек и **английского**:

- максимум **2 пропуска в календарную неделю** (пн–вс) на каждую привычку отдельно;
- пропуск — явная кнопка «Пропустить сегодня» до 23:59;
- статус дня: `skipped` (⚪ серый в календаре);
- пропуск **не увеличивает** и **не уменьшает** цель;
- 3-й пропуск в ту же неделю — **недоступен**: только выполнить или получить провал в 23:59.

**Без пропусков (строгий режим):** все привычки тёмной стороны.

**При активном залоге** на привычку пропуски **запрещены** — любой пропуск = провал залога.

## 4. Онбординг

Сценарий **«Новая глава» v2.0** — не анкета, а путешествие. Эмоциональная дуга: приветствие → битва (тёмная сторона) → триумф. Каждый шаг — маленькая победа. Без иллюстраций и фото: только текст, карточки и поля.

**Шаг 0. Приветствие**

- Заголовок: «Привет, воин!»
- Подзаголовок: «Я — твой навигатор. Давай выясним, кто ты сейчас и куда хочешь прийти. Это займёт всего 3 минуты, но изменит твою жизнь на годы вперёд. Готов?»
- Кнопка: «Да, погнали!»

**Шаг 1. Кто ты? (регистрация)**

- Текст: «Чтобы я подобрал тебе идеальный план, давай познакомимся поближе. Твои ответы останутся строго между нами.»
- Поля: имя, email, пароль, возраст, пол.
- Если пользователь уже авторизован — шаг пропускается.

**Шаг 2. Светлая сторона ☀️**

- Заголовок: «Что ты хочешь прокачать в себе? ☀️»
- Подзаголовок: «Выбери то, что зажигает тебя. Это будут твои новые суперсилы. Добавь свои, если хочешь.»
- Карточки: читать книги, отжиматься, бегать, планка, «+ Своё занятие».
- Под каждым выбранным — «Сколько (страниц/раз/минут) ты делаешь это прямо сейчас?»
- Речь после выбора: «Отлично! Ты уже делаешь N …? Мы начнём с этого уровня и будем расти каждый день. Светлая сторона гордится тобой.»

**Шаг 3. Тёмная сторона 🌑**

- Заголовок: «Что тянет тебя на дно? 🌑»
- Подзаголовок: «Пришло время отрубить хвосты. Выбери то, с чем ты хочешь покончить навсегда. Я буду рядом, но рубить придётся тебе.»
- Карточки: курение, сахар, сладости, соцсети, грызть ногти.
- Вопрос baseline для каждой (кроме «грызть ногти» — сразу режим отказа).
- Речь: «Хорошо. Ты … N в день. Мы будем снижать по 1 в день. Тёмная сторона берёт тебя под контроль.»

Максимум **6** привычек суммарно (светлая + тёмная).

**Шаг 4. Твоя реальность (режим и тело)**

- Заголовок: «Давай заложим фундамент.»
- Подзаголовок: «Ответь на несколько вопросов, чтобы я подстроил нагрузку под твой реальный день, а не под абстрактный идеал.»
- Поля: вес, рост, время подъёма, время сна, ползунок свободного времени (15–180 мин).
- Превью: «Твой план на день займёт около N минут. Это реально?»

**Шаг 5. Характер помощника**

- Заголовок: «Как ты хочешь, чтобы я с тобой разговаривал?»
- Карточки: 🌿 Мягкий / 📐 Строгий / 🗡️ Сержант (harshness 1–3).

**Шаг 6. Финальный аккорд**

- Заголовок: «Это твой момент.»
- Переключатель «Учить английский? (+5 мин/день)».
- Тарифы: 3 дня бесплатно → 1990 ₽ / 3790 ₽ / 5490 ₽.
- Финальная речь + кнопка «Начать новую главу».
- На submit: PATCH `/me` (тело + harshness) → PATCH `/english/settings` → POST `/habits` для каждой выбранной привычки.

## 6. Главные экраны

### 6.1. Светлая сторона ☀️

Светлый фон, акценты зелёного.

**Верх экрана:**

1. Приветствие: «Привет, {имя}!»
2. **Полоска недели** — горизонтальный календарь (Пн–Вс), активный день выделен кругом + цветная точка (🟢🟡🔴⚪)
3. **4 карточки статистики:**
   - Выполнено сегодня
   - Срывы за неделю
   - Минут / 🍅 за день
   - Серия дней

**Полоска недели** — быстрый выбор дня. Тап по дню → чекины за этот день. Свайп влево/вправо → другая неделя.

Для каждой полезной привычки — **карточка с ползунком** и **помодоро**:

```
💻 Программирование — цель: 30 мин
Помодоро: [▶ Старт 25:00]  ·  Сегодня: 2 🍅 (50 мин)
[━━━━━━━●━━] 50 мин
[Сохранить вручную]  [Пропустить сегодня]
```

- **Ползунок** — ручной ввод (страницы, минуты, повторы).
- **Помодоро** — таймер фокуса (по умолчанию 25 мин работа + 5 мин перерыв).
- Завершённый помодоро **автоматически** добавляет минуты к прогрессу привычки.
- Линия цели видна на ползунке.
- value ≥ цель → 🟢 выполнено.
- Внизу: «Сегодня: 32 из 45 мин» (прогресс дневного бюджета).

### 6.2. Помодоро-таймер

Встроен в **светлую сторону** для привычек с единицей «минуты» (шаблоны и свои).

| Параметр | Значение по умолчанию |
|----------|----------------------|
| Работа | 25 мин |
| Перерыв | 5 мин |
| Длинный перерыв | 15 мин (каждые 4 помодоро) |

**Flow:**

1. Пользователь нажимает **«Старт помодоро»** на карточке привычки.
2. Экран фокуса: обратный отсчёт, название привычки, кнопки «Пауза» / «Стоп».
3. По окончании 25 мин → 🍅 +1, минуты засчитываются в привычку, push «Перерыв 5 мин».
4. Перерыв → можно запустить следующий помодоро.
5. Несколько помодоро в день **суммируются** к цели.

**Настройки (профиль):**

- Длительность работы: 15 / 25 / 45 / 50 мин
- Длительность перерыва: 5 / 10 / 15 мин

**Технически:**

- Таймер работает в браузере (Web Worker), не сбрасывается при сворачивании PWA.
- Push по окончании помодоро и перерыва (если разрешены).
- Офлайн: помодоро считается локально, синхронизируется при появлении сети.

### 6.3. Тёмная сторона 🌑

Тёмный фон, акценты красного. **Полоска недели** и **карточки статистики** — как на светлой (те же компоненты, тёмная тема).

**Привычки с лимитом** (курение, сахар, сладости, соцсети) — **ползунок** «сколько сегодня» + линия лимита:

```
🚬 Курение — лимит: ≤ 18 сигарет (было 20)
Сегодня: [━━━━●━━] 16 сигарет
[Сохранить]
Цель завтра при успехе: 17
```

**Привычки полного отказа** (ногти, курение на фазе `abstinence`) — **таймер**:

```
🚬 Курение — не курю
Не курю: 3 дня 14:22:08
[Я сорвался]
```

Для соцсетей и «тупых дел» (TikTok, Reels, Shorts и т.п.):

```
📱 Тупые дела
Сегодня: 8 из 30 мин
[Начал смотреть]  ← запускает сессию 15 мин
```

- Кнопка **«Начал смотреть»** — пользователь честно отмечает, что залипает.
- Стартует **сессия 15 минут** (обратный отсчёт).
- На время сессии доступна кнопка **«Закончил раньше»** — завершает сессию досрочно и засчитывает только фактическое время.
- Сразу при нажатии → **push** (предупреждение по уровню жёсткости).
- По истечении 15 мин → **push** «время вышло, отложи телефон».
- Минуты сессии суммируются в дневной лимит (30 мин).
- Превысил лимит → день = провал.

Для **ногтей** на фазе `abstinence` — только таймер и «Сорвался».

### 6.4. Блок «Залог»

- сумма: **5000 ₽** за привычку;
- кнопка «Активировать залог»;
- таймер до окончания 30-дневного цикла;
- статус: «Активен» / «Успешно» / «Провален».

### 6.5. Блок «Статистика»

Мини-дашборд на светлой и тёмной стороне (одинаковые компоненты, разная тема):

- **мини-график за 7 дней** — линейный или столбчатый, **живой** (пересчитывается после каждого чекина);
- процент успешных дней по каждой привычке;
- количество срывов за месяц;
- кнопка **«Подробнее»** → полноэкранные графики за месяц (см. раздел 27).

Подробные типы графиков, интерактивность и API — в **разделе 27**.

## 7. Раздел «Английский»

Опциональный модуль для изучения языка в рамках ежедневного саморазвития.

### 7.1. Активация

- включается на онбординге (шаг 4) или в профиле;
- после включения появляется вкладка «Английский».

### 7.2. Ежедневный урок

- каждый день — **одно новое видео** (последовательная программа: день 1, день 2, …);
- видео хранятся на сервере / CDN (URL в БД);
- длительность: 5–15 минут (MVP);
- после просмотра (≥ 80% длительности) — кнопка «Урок пройден».

### 7.3. Логика

| Действие           | Результат                                        |
| ------------------ | ------------------------------------------------ |
| Урок пройден       | День засчитан, переход к следующему видео завтра |
| Пропуск            | До 2 раз в неделю (как у полезных привычек)      |
| Не открыл до 23:59 | Провал дня (урок остаётся тем же завтра)         |

Пропуск **не продвигает** номер урока — пользователь продолжает с текущего видео.

### 7.4. Интерфейс вкладки

- текущий урок: название, номер дня, плеер;
- прогресс: «День 12 из программы»;
- кнопки: «Урок пройден» / «Пропустить сегодня»;
- архив пройденных уроков (список).

## 8. Ежедневная логика

### 8.1. Тёмная сторона

**Лимитные привычки** (`limit`, фаза `reduction`) — курение, сахар, сладости, соцсети:

- Пользователь вводит **сколько сделал сегодня** (ползунок).
- value ≤ `current_goal` → успех → завтра `goal -= step`.
- value > `current_goal` → провал, goal не меняется.
- Не отметился до 23:59 → провал.

**Курение:** при `current_goal = 0` → переход в фазу `abstinence` (таймер, кнопка «Сорвался»).

**Полный отказ** (`abstinence`) — грызть ногти, курение на фазе `abstinence`:

- Таймер с `last_relapse_at`.
- «Я сорвался» → провал, таймер сбрасывается.
- До 23:59 не сорвался → успех (автозакрытие).

**Соцсети / «тупые дела»:**

- Кнопка «Начал смотреть» → сессия 15 мин, push при старте и по окончании.
- Кнопка «Закончил раньше» → завершает сессию досрочно, учитывается фактическое время.
- Учёт минут в дневном лимите (30 мин).
- Лимит не превышен до 23:59 → успех; превышен → провал.

### 8.2. Светлая сторона (ползунки)

- Пользователь двигает **ползунок** до своего результата и сохраняет.
- value ≥ цели → день выполнен, цель растёт завтра.
- value < цели → день провален.
- «Пропустить» → `skipped` (до 2 раз в неделю).
- Не отметился до 23:59 → провал.

### 8.3. Автозакрытие дня

Фоновый worker **каждую минуту** (см. §9.7) проверяет пользователей, у которых наступила полночь (23:59 в timezone пользователя), и закрывает незавершённые чекины по правилам выше.

**Чёткое правило для `abstinence`:** если за день не было «Сорвался» → в 23:59 `success`.

**Чёткое правило для `limit` (reduction):** если чекин не создан до 23:59 → `fail`. Если value ≤ goal → `success`.

**Чёткое правило для соцсетей:** дополнительно учёт сессий «тупых дел» и дневного лимита минут.

### 8.4. Залоговая механика

| Параметр       | Значение                                                     |
| -------------- | ------------------------------------------------------------ |
| Сумма          | 5000 ₽ за привычку                                           |
| Срок           | 30 дней                                                      |
| Условие успеха | Ни одного провала за 30 дней (пропуски при залоге запрещены) |
| Успех          | Возврат 5000 ₽ + бейдж «Стальной характер»                   |
| Провал         | 5000 ₽ → выбранный благотворительный фонд                    |
| Ограничение    | 1 активный залог в месяц                                     |

**Благотворительные фонды (на выбор пользователя):**

1. Онкологический фонд
2. Детский фонд
3. Фонд защиты животных

**Техническая модель платежа:** списание 5000 ₽ при активации → возврат при успехе. Не «холд на 30 дней» (платёжные системы не поддерживают длительный холд).

**Важно про пропуски при залоге:**

- При активном залоге кнопка «Пропустить» для этой привычки **блокируется**.
- Перед активацией залога показываем предупреждение: «Внимание: пропуски при залоге невозможны».

**Форс-мажор (болезнь/травма):**

- В v1 поддержки форс-мажора нет (кроме предупреждения).
- В v1.1 допускается обращение в поддержку и ручное закрытие залога администратором с возвратом средств.

## 9. Уведомления (Push)

### 9.1. Ежедневное расписание (3 push)

По времени пользователя:

```
утро  = wake_time
день  = wake_time + (sleep_time - wake_time) / 2
вечер = sleep_time - 30 минут
```

Если интервал между подъёмом и сном < 6 часов — только утро и вечер.

### 9.2. Подбадривание некурящих (3–5 push в день)

Если у пользователя курение в фазе `abstinence` (или `current_goal = 0`) и сегодня **не было срыва**:

- дополнительно **3–5 push** в течение дня;
- равномерно между подъёмом и сном (не чаще 1 раза в 2 часа);
- тексты по уровню жёсткости, фокус: «ты на правильном пути», «уже N часов/дней без сигареты».

Примеры:

| Жёсткость | Пример                                               |
| --------- | ---------------------------------------------------- |
| Мягкий    | «Ты уже 2 дня без сигарет. Горжусь тобой!»           |
| Строгий   | «Два дня без курева. Не расслабляйся.»               |
| Сержант   | «48 часов без дыма. Так держать, не ссы, прорвёмся!» |

Пуши **не отправляются**, если пользователь сегодня нажал «Сорвался».

### 9.3. «Тупые дела» — сессия соцсетей

| Событие                       | Push                                            |
| ----------------------------- | ----------------------------------------------- |
| Нажал «Начал смотреть»        | Сразу: «15 минут. Потом — стоп.» (по жёсткости) |
| Истекли 15 минут сессии       | «Время вышло. Отложи телефон.»                  |
| Нажал «Закончил раньше»       | (без push)                                      |
| Превышен дневной лимит 30 мин | «Лимит на сегодня исчерпан.»                    |

### 9.4. Шаблоны по уровню жёсткости (расписание)

| Время | Мягкий                                              | Строгий                             | Сержант                                                   |
| ----- | --------------------------------------------------- | ----------------------------------- | --------------------------------------------------------- |
| Утро  | «Доброе утро! Сегодня отличный день, чтобы начать!» | «Подъём! Не забывай про свои цели.» | «Встал? Отлично. Сегодня не ссы, а делай.»                |
| День  | «Как дела? Ты справляешься!»                        | «Проверь себя. Ты выполнил план?»   | «Что там по привычкам? Не тормози!»                       |
| Вечер | «Осталось немного. Отметь сегодняшний день.»        | «Не затягивай. Отметь сейчас.»      | «Последний шанс. Если не отметишь — завтра будет стыдно.» |

### 9.5. Мгновенные уведомления

- «Я сорвался» (курение, ногти) — сообщение по жёсткости.
- Ползунок ≥ цели (светлая сторона) — похвала по жёсткости.

### 9.6. Технические требования

- Web Push API (VAPID-ключи).
- На iOS — push только после установки PWA («На экран Домой»).
- Шаблоны в БД: `event_type × harshness_level`.
- Типы событий: `morning`, `afternoon`, `evening`, `relapse`, `success`, `smoke_cheer`, `doom_scroll_start`, `doom_scroll_end`, `doom_scroll_limit`.
- Сессии «тупых дел» — отложенные push через BullMQ (через 15 мин после старта); **одна job на сессию**.
- Расписание (утро / день / вечер / cheer) — не чаще 1 раза за событие в локальный день пользователя; учёт в `push_delivery_log` (§17).

### 9.7. Фоновые задачи (обязательная реализация)

Worker/cron (BullMQ) запускается **каждую минуту** и проверяет:

- истёк ли таймер любой активной сессии «тупых дел» → отправить push окончания;
- наступило ли 23:59 в timezone пользователя → автозакрытие дня;
- наступило ли время утро/день/вечер → отправить push;
- нужно ли отправить подбадривание некурящим (3–5/день) → отправить;
- истёк ли 30-дневный залог → завершить (success/failed).

## 13. PWA и офлайн-режим

- `manifest.json` (иконки, тема, `display: standalone`).

## 15. Безопасность и юридическая часть

### 15.1. Техническая безопасность

- HTTPS на всех соединениях.
- Пароли: bcrypt или argon2.
- JWT: access (15 мин) + refresh (30 дней).
- PostgreSQL encryption at-rest + TLS.
- Rate limiting на API.
- Валидация всех входных данных (Zod).

### 15.3. Психологическая безопасность («Красная кнопка»)

В профиле добавить переключатель **«Режим тишины»**:

- отключает все push-уведомления на **24 часа**;
- уровень жёсткости принудительно становится «Мягкий» на эти 24 часа;
- залог **не сгорает**: день засчитывается как `skipped` (без штрафа);
- ограничение: активировать можно не чаще **1 раза в 30 дней**.

### 15.2. Юридические документы

Перед использованием пользователь принимает:

1. **Пользовательское соглашение:**
   - Залог — добровольное пожертвование при нарушении условий.
   - Сервис не является медицинским учреждением и не лечит зависимости.
   - Ответственность за психическое состояние — на пользователе.
   - Возврат денег возможен только при успешном выполнении условий залога.

2. **Политика конфиденциальности:**
   - Соответствие 152-ФЗ (РФ).
   - Данные о привычках и зависимостях — чувствительные, требуется явное согласие.
   - Данные не передаются третьим лицам.
   - Право на удаление и экспорт данных.

3. **Дисклеймер:** рекомендации по нагрузке ориентировочные, не являются медицинской консультацией.

## 23. Геймификация и нарратив (v1.1 — «вау-эффект»)

> Не блокирует релиз v1.0, но критично для удержания и эмоциональной связи. Структуру БД и API закладываем сразу, UI/анимации — после MVP.

Три кита эмоциональной связи: **Сущности**, **Ранги**, **Красный Босс**.

### 23.1. Светлая и Тёмная сторона — сущности с характером

Стороны — не просто вкладки, а **союзники** с голосом.

| Сторона    | Голос            | Тон                        | Где звучит                      |
| ---------- | ---------------- | -------------------------- | ------------------------------- |
| Светлая ☀️ | «Твой потенциал» | Зелёный текст, вдохновение | Онбординг, успехи, ранги, push  |
| Тёмная 🌑  | «Твой контроль»  | Красный текст, дисциплина  | Онбординг, срывы, таймеры, Босс |

**Правила UI:**

- Заголовки вкладок сопровождаются короткой репликой сущности (1 строка, меняется раз в день).
- Push от светлой стороны — поддержка; от тёмной — контроль и предупреждения.
- Тексты сущностей не дублируют уровень жёсткости — жёсткость модифицирует формулировки поверх голоса сущности.

### 23.2. Ранги привычек (вместо «просто цифр»)

У каждой привычки — **ранг** по дням успеха (серия `success` подряд). Отображается на карточке вместе с целью.

**Светлая сторона** (спорт, книги, планка…):

| Ранг    | Дней успеха подряд |
| ------- | ------------------ |
| Новичок | 0–7                |
| Боец    | 8–21               |
| Мастер  | 22–50              |
| Легенда | 51+                |

**Тёмная сторона** (курение, ногти…):

| Ранг          | Дней без срыва подряд |
| ------------- | --------------------- |
| Срываюсь      | 0–3                   |
| Держусь       | 4–14                  |
| Стальной      | 15–30                 |
| Неприкасаемый | 31+                   |

**Механика:**

- Срыв / провал → ранг пересчитывается (серия обнуляется).
- Пропуск (`skipped`) → серия **не прерывается**, ранг не падает.
- При повышении ранга → **отдельный push** с поздравлением + краткая анимация на карточке.
- Ранг хранится в `habits.current_rank` (кеш) и пересчитывается из `checkins`.

**Пример push:** «Ты — Боец в отжиманиях. 14 дней подряд. Светлая сторона гордится.»

### 23.3. «Тупые дела» — Красный Босс

Блок соцсетей — не счётчик, а **босс**, с которым борется Тёмная сторона.

**UI карточки:**

```
👹 Красный Босс
Лимит: 30 мин · Жизни: ●●●○○ (3 из 5)
[Начал смотреть]  [Закончил раньше]
```

**Механика жизней:**

- Старт дня: **5 жизней**.
- Каждые **10 минут сверх лимита** (30 мин) → −1 жизнь.
  - 30 мин — ок; 31–40 мин → −1; 41–50 → −2; и т.д.
- **0 жизней** → новые сессии заблокированы на **60 минут** (экран затемнён: «Ты проиграл боссу. Перерыв 60 минут.»).
- **Уложился в лимит до 23:59** → Босс побеждён: конфетти в интерфейсе + реплика Тёмной стороны.

**Связь с чекином:**

- Лимит не превышен → `success` (как в п. 8.3).
- Лимит превышен или жизни = 0 → `fail`.
- Блокировка сессий не отменяет уже идущую сессию, но запрещает старт новой.

### 23.4. Техническая реализация (задел)

```sql
-- Ранг привычки (кеш, пересчитывается из checkins)
ALTER TABLE habits ADD COLUMN current_rank VARCHAR NOT NULL DEFAULT 'novice';
ALTER TABLE habits ADD COLUMN streak_days INTEGER NOT NULL DEFAULT 0;

-- Состояние Красного Босса на день
CREATE TABLE doom_boss_daily (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id      UUID REFERENCES users(id) ON DELETE CASCADE,
    habit_id     UUID REFERENCES habits(id) ON DELETE CASCADE,
    date         DATE NOT NULL,
    lives_left   INTEGER NOT NULL DEFAULT 5,
    minutes_used INTEGER NOT NULL DEFAULT 0,
    blocked_until TIMESTAMP,
    boss_defeated BOOLEAN DEFAULT FALSE,
    UNIQUE (user_id, habit_id, date)
);
```

**API (v1.1):**

| Метод | Путь               | Описание                            |
| ----- | ------------------ | ----------------------------------- |
| GET   | `/habits/:id/rank` | Текущий ранг и серия                |
| GET   | `/habits/:id/boss` | Состояние Красного Босса на сегодня |

**Push:** новый тип события `rank_up` в `notification_templates`.

## 24. Обработка ошибок и граничные случаи

> Справочник для разработки: как система ведёт себя при сбоях и нестандартных ситуациях.

### 24.1. Push не доставлен

Push-уведомления **не гарантируются** (зависит от браузера, ОС, настроек, установки PWA).

**Поведение:**

- В интерфейсе всегда виден индикатор: «Последний чекин: сегодня в 14:32».
- Если push не пришёл — пользователь открывает приложение и отмечает привычку вручную.
- В профиле — кнопка **«Проверить push»** (отправка тестового уведомления).

**API:** `POST /push/test` — тестовый push текущему пользователю.

### 24.2. Ошибки платежей (ЮKassa)

**Подписка — списание не прошло** (только тариф `monthly` с автопродлением):

- Статус подписки → `past_due`.
- Push пользователю: «Не удалось списать оплату. Проверьте карту.»
- Повторная попытка через **3 дня**, максимум **3 попытки**.
- После 3 неудач → статус `expired`, доступ к платным функциям закрыт.

**Залог — списание не прошло:**

- Залог **не активируется**.
- UI: «Платёж не прошёл. Проверьте карту.»

**Залог — возврат не прошёл (успешное завершение):**

- Залог помечается `failed` с флагом `refund_error = true`.
- Алерт администратору в **Sentry**.
- Пользователю: «Возврат в обработке. Мы свяжемся с вами.»

**Схема БД (дополнение):**

```sql
-- subscriptions.status: добавить 'past_due'
-- pledges: добавить refund_error BOOLEAN DEFAULT FALSE
```

### 24.3. Конфликт офлайн-синхронизации

- **Сервер — источник истины.**
- При конфликте (клиент прислал запись с более старым `updated_at`, чем на сервере) → **409 Conflict**, клиент перезапрашивает актуальные данные.
- Если пользователь был офлайн несколько дней → синхронизация **пачкой**: `POST /checkins/batch` (массив чекинов, каждый валидируется отдельно; конфликтные — в ответе с кодом ошибки).

### 24.4. Удаление аккаунта при активном залоге

Перед удалением проверяем наличие активного залога (`status = 'active'`).

1. Показываем предупреждение: «У вас активен залог. При удалении аккаунта залог будет завершён как провал, средства уйдут в фонд.»
2. Пользователь подтверждает → залог → `failed`, аккаунт удаляется.
3. Отмена → удаление не выполняется.

### 24.5. Браузер не поддерживает PWA

| Платформа                    | Поведение                                                            |
| ---------------------------- | -------------------------------------------------------------------- |
| iOS < 16.4                   | Сообщение: «Обновите iOS до 16.4+ или используйте Chrome на Android» |
| Десктоп без PWA              | Работает как обычный сайт: без офлайн и без push                     |
| Android / iOS 16.4+ / Chrome | Полный функционал PWA                                                |

### 24.6. Пользователь удалил PWA с рабочего стола

- Можно установить заново по той же ссылке.
- Данные на **сервере** — прогресс не теряется.
- Push-подписка создаётся **заново** при повторной установке и разрешении уведомлений.

### 24.7. Смена часового пояса

- Все даты чекинов хранятся в **UTC** на сервере.
- При смене `timezone` в профиле:
  - пересчитывается расписание push (утро / день / вечер / cheer);
  - **даты уже созданных чекинов не пересчитываются**;
  - новая timezone применяется **со следующего дня** (автозакрытие 23:59, новые чекины).

## 27. Визуализация прогресса (живые графики)

Критично для удержания. Пользователь должен видеть **траекторию** изменений — графики не статичные картинки, а пересчитываются от действий пользователя в реальном времени.

### 27.1. Типы графиков

Графики отображаются в блоке «Статистика» на каждой стороне (светлая / тёмная) и на отдельной вкладке (v1.1).

**Светлая сторона (рост):**

| График | Тип | Данные |
| ------ | --- | ------ |
| «Прогресс цели» | Линейный (daily) | Изменение `current_goal` за последние 30 дней — видно, как планка поднимается |
| «Выполнение vs Цель» | Столбцы + линия | Синие столбцы (выполнено), зелёная линия (цель) — видно, когда пользователь перебивает планку |

**Тёмная сторона (снижение / отказ):**

| График | Тип | Данные |
| ------ | --- | ------ |
| «Снижение» | Линейный (daily) | Для курения / сахара / сладостей / соцсетей: изменение `current_goal` (лимит) за 30 дней — линия идёт вниз |
| «Таймер отказа» | Линейный (рост) | Для `abstinence`: дни подряд без срыва |
| «Тупые дела» | Столбцы + линия | Столбцы — минуты в день; красная линия — лимит (30 мин) |

**Общие графики:**

| График | Тип | Примечание |
| ------ | --- | ---------- |
| Сводка за месяц | Тепловая карта (календарь) | Уже в п. 10.2; дополнить числовыми значениями при наведении |

### 27.2. Интерактивность

Графики **живые**, не статичные:

- при переключении периода (неделя / месяц / квартал) — данные перерисовываются;
- при клике на точку / столбец — tooltip: дата, значение, статус дня (`success` / `fail` / `skipped`);
- анимация роста / падения при загрузке — ощущение динамики;
- библиотека: **Recharts** (уже в стеке, п. 16.1): `LineChart`, `BarChart`, `PieChart`, адаптивная ширина.

### 27.4. Обновление данных

**Сервер (backend):**

- при закрытии дня (worker, 23:59) в `goal_snapshots` сохраняется `goal_value` за этот день — **до** прогрессии цели на завтра;
- в `daily_stats` сохраняется итог дня по каждой привычке (`status`, `value`, `minutes_total`);
- эндпоинты `/stats/*` читают `goal_snapshots` и `daily_stats`, а не пересчитывают историю из текущего `current_goal`.

**Клиент (PWA):**

- графики обновляются после каждого чекина (инвалидация TanStack Query);
- данные кешируются на клиенте, синхронизируются после `POST /checkins/batch`.

### 27.5. UI-референс

- мини-график в блоке «Статистика» — высота ~120 px, без оси X (только тренд).
